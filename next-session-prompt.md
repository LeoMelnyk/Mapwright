## Asset bundles (new since batch 8)

All three asset catalogs (props, themes, textures) are now served as
single-file bundles for a one-fetch startup:

- `src/props/bundle.json` — regenerated by `tools/update-manifest.js`
  alongside `manifest.json`. **Run update-manifest.js after ANY prop
  change**; otherwise the editor keeps serving the stale bundle. The
  script writes both files in one pass.
- `src/themes/bundle.json` — regenerated by
  `tools/update-themes-manifest.js` (new). Only matters if you also
  author themes.
- `<userData>/textures/bundle.json` — regenerated automatically by the
  server's `writeManifest()` whenever a texture download completes.
  Don't touch manually unless a texture is missing from the bundle.

Each bundle has a content-hash version that changes when the underlying
files change, so the browser's HTTP cache busts correctly on reload.
localStorage is no longer used to cache any catalog — HTTP ETag
revalidation is enough.

**If the editor still shows stale props after you author new files,
you forgot to run `update-manifest.js`.** That regenerates the bundle
the client actually reads.

## Hard-won lessons (don't relearn)

- **Rooms are ALREADY wired for missing props.** Every name in the
  report is referenced by a vocab spec. Author the prop, don't edit
  specs. The 3 Tier-1 aliases (counter, bookshelf-short,
  rat-swarm-marker) are the exception — those need spec updates.

- **Tall vertical props are the biggest perspective trap.** If an
  agent's brief says "tall pillar of X, footprint 2x1, draw from above"
  it will STILL draw a side-elevation shaft running north-south through
  the footprint. Give them `1x1` or `2x2` footprints and explicit
  instruction: "radial cross-section seen from directly above, decorate
  with radial spokes + center highlight + outer halo — NEVER a vertical
  shaft crossing the footprint." See `obelisk.prop`, `pillar.prop`,
  `flame-pillar.prop`, `basalt-pillar.prop`, `pearl-obelisk.prop` for
  canonical examples.

- **Light-emitting props need `lights:` frontmatter in YAML list form.**
  Reference `crystal-glow.prop` for the exact format. Agents sometimes
  emit `lights: candle 3 #ff7030 0.3` (parses as a bare string, fails
  server-side). Flag this explicitly in the prompt.

- **The user iterates on visual quality.** Expect 2–3 revision rounds
  on specific props the user flags by eye. Don't defend — redraw. See
  batch-7 follow-up for the pattern (bubble-pillar, coral-pillar,
  kelp-frond-giant, kelp-strand, pearl-obelisk, silver-cord-pylon were
  all redrawn after the user flagged them as "wrong perspective").

- **bridge-wood and rope-bridge were deleted** — the editor has a
  first-class bridge tool, no prop needed. If an agent is about to
  author a prop that duplicates an editor tool (bridges, stairs, walls,
  doors, textures), stop and confirm with the user first.

- **Don't forget to run `update-manifest.js`.** It writes both
  `manifest.json` and `bundle.json`; the editor reads the bundle.
  Authoring a .prop file without running this = invisible prop.

- The user gives full creative freedom on visual style and wants
  quality over speed. Don't ask permission to invent — just match
  neighboring prop quality and ship.

- Agents are the preferred mode; don't burn main-context tokens
  hand-authoring prop files unless the user is actively iterating.
